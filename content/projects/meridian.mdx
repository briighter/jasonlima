Engineering teams move slower over time. Not because developers get lazier —
because the infrastructure around them accumulates friction.

Meridian makes that friction visible.

## What it tracks

- CI build duration trends by branch, author, and time of day
- Flaky test detection (tests that pass and fail on the same commit)
- PR cycle time from first commit to merge
- Deployment frequency and mean time to restore

## Architecture

The core is a webhook receiver that ingests GitHub Actions events, processes them
through a transform pipeline, and writes to PostgreSQL. A separate read API handles
the dashboard queries with heavy use of window functions and materialized views
for aggregations.

The frontend is a React SPA with a custom charting layer built on top of D3 — no
off-the-shelf chart library was flexible enough for the interaction model I wanted.
